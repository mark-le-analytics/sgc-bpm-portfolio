# SGC Business Process Transformation  
**Business Process Analysis · BPMN Modelling · Automation Design**


## Overview

This project presents an end-to-end analysis and redesign of the coaching operations at **Swinburne Guru Consulting (SGC)**.

The work demonstrates a full **Business Process Management (BPM) lifecycle**, including:

- Business problem identification  
- As-Is process analysis  
- BPMN modelling (Level 2 diagrams)  
- To-Be process redesign  
- Automation strategy (BPMS integration)  

The objective is to improve operational efficiency, reduce delays, and create a scalable process.



## Business Context

SGC is a coaching services business that connects clients with suitable coaches and manages the full assignment lifecycle.

The current process contains several inefficiencies:

- Manual client onboarding and data entry  
- Delays in coach assignment and scheduling  
- Rework loops caused by availability conflicts  
- Late payment processing and cash flow issues

## Phase 1 — Business Problem

Defines the operational context and key challenges faced by SGC, including inefficiencies in onboarding, scheduling, and process coordination.



## Phase 2 — Process Analysis (As-Is)

Identifies critical issues in the current process:

- Inefficient coach selection (manual and sequential)  
- Redundant communication between roles  
- Scheduling conflicts causing process loops  
- Dependency on client feedback delaying completion  

These issues increase workload, delay service delivery, and reduce client satisfaction.



## Phase 3 — Process Modelling (BPMN)

### As-Is Process

The current workflow is modelled using BPMN across multiple subprocesses:

- Create Assignment  
- Recruit / Find Coach  
- Schedule Assignment  
- Verify Assignment  
- Collect Feedback  
- Process Payment  

These models highlight bottlenecks, delays, and unnecessary handoffs.



### To-Be Process (Redesigned)

The improved process introduces key changes:

- **Digital Client Intake** → eliminates manual data entry  
- **Centralised Coach Availability** → reduces scheduling delays  
- **Early Payment Processing** → improves cash flow  
- **Automated Feedback Handling** → removes process bottlenecks  
- **Structured Exception Handling** → consistent refund decisions  

These improvements streamline operations and remove rework loops.



## Automation Design (BPMS)

The redesigned process is extended using a **Business Process Management System (BPMS)**.

### Automated Activities

- Client record validation and creation  
- Missing information detection  
- Discount rule application  
- Automated notifications and updates  

### Impact

- Reduced manual workload  
- Faster processing time  
- Improved data accuracy  
- Real-time workflow control  



## Business Impact

The redesigned solution delivers:

- Faster assignment completion  
- Reduced scheduling delays  
- Lower administrative workload  
- Improved client experience  
- Scalable process design
